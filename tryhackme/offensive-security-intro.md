# Writeup — Offensive Security Intro

**Plataforma:** TryHackMe  
**Dificuldade:** Fácil  
**Categoria:** Segurança Ofensiva  
**Data de conclusão:** Abril/2025  
**Pontos ganhos:** 32  

---

## O que é essa sala

Introdução ao conceito de segurança ofensiva — a prática de pensar como um atacante para encontrar vulnerabilidades antes que pessoas mal-intencionadas encontrem. A sala simula o hack ético de um banco fictício (FakeBank) para demonstrar como funciona o trabalho de um pentester.

---

## O que aprendi

- O que é segurança ofensiva e como ela se diferencia da segurança defensiva
- Por que empresas contratam hackers éticos (pentesters)
- Como funciona um teste de invasão (pentest) na prática
- Primeiro contato com terminal Linux em ambiente controlado
- Conceito de que entender como atacar é fundamental para saber defender

---

## Ferramentas utilizadas

- Terminal Linux
- Navegador web
- GoBuster (ferramenta de enumeração de diretórios web)

---

## Resumo do desafio

A sala apresentou o conceito de hacking ético e colocou o usuário para praticar invadindo um banco fictício de forma legal. O objetivo era encontrar uma página escondida no site do FakeBank usando enumeração de diretórios, acessar a conta e transferir dinheiro para demonstrar a vulnerabilidade.

---

## Dificuldades encontradas

Primeira vez usando o terminal Linux para rodar uma ferramenta de segurança. Levou um tempo para entender a sintaxe do comando e interpretar o output da ferramenta.

---

## Principais aprendizados

A segurança ofensiva não é sobre causar dano — é sobre encontrar problemas antes que alguém mal-intencionado encontre. Profissionais dessa área são contratados e pagos para fazer exatamente isso de forma ética e autorizada.
